## Key Concepts

### The 10-Step Routine

Every client follows the same path:

1. **Create case ID** — `KFS-YYYY-###-ClientName`
2. **Send quote** — Customize message template
3. **Collect agreement + payment** — $800 or $1,600
4. **Get intake** — Client fills form
5. **Verify** — Checklist against requirements
6. **Submit formation** — LLC to state
7. **Track approvals** — Monitor state + IRS
8. **Registrations** — EIN, NJ-REG, BRC
9. **Banking package** — Docs + checklists
10. **Delivery** — Client acknowledges + close

---

### Status Vocabulary

Use **only these** statuses (see `docs/03_HERMES_GITHUB_OBSIDIAN_PROTOCOL.md`):

```
LEAD → QUOTED → PAID_DEPOSIT → INTAKE_PENDING → INTAKE_RECEIVED 
→ VERIFYING → FORMATION_SUBMITTED → FORMATION_APPROVED 
→ REGISTRATION → BANK_READY → DELIVERED → CLOSED
```

(Optional: `BLOCKED` for stuck cases)

---

### Sensitive Data Rules

**GitHub:** SOPs, templates, sanitized examples only (no client data)  
**Obsidian:** Case notes, decisions, task lists + references to secure storage  
**Secure Storage:** SSNs, driver's licenses, bank data, signatures (off-vault)  
**Change Log:** Every change logged + approved

See `docs/03_HERMES_GITHUB_OBSIDIAN_PROTOCOL.md` for details.

## FAQ

**Q: Can I change the workflow steps?**  
A: The 10-step routine is tested. If a step seems unnecessary, ask Jayto.

**Q: What if the client is doing something regulated?**  
A: Escalate to Jayto. These may need special licensing.

**Q: Where do I store client documents?**  
A: Off-vault in secure storage. Reference them in the case file (e.g., "Tax return: CRC vault #ABC123").

**Q: Can I commit client data to GitHub?**  
A: No. Only SOPs, templates, automation, and anonymized examples.

**Q: What's the timeline per client?**  
A: 3–6 weeks from quote to delivery (depends on state processing + client responsiveness).

**Q: Can I customize the templates?**  
A: Yes. Update them in the `templates/` folder, then test with a case.
